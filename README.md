# CUMT-Net-AutoLogin

矿大（CUMT）校园网无感自动登录脚本。

**核心优势**：只需配置一次，之后每次电脑连接校园网（WIFI `CUMT_Stu` / `CUMT_Tec` 或有线网）时，系统都会在底层瞬间自动完成登录。与其他每次开机或断网都需要手动点一下执行的工具不同，它能让你彻底实现“一劳永逸”，完全忘记网页认证这回事。

> ⚠️ **注意**：本项目利用了 Windows 系统自带的任务计划功能，**仅支持 Windows 操作系统**。

---

## 🚀 安装指南

### 1. 准备文件
1. 点击本页面绿色的 `<> Code` 按钮，选择 **Download ZIP** 下载并解压。
2. ⚠️ **极其重要**：请将解压后的文件夹移动到一个**你平时不会随手删除的目录**（例如 `D:\Tools\CUMT_Net_AutoLogin\`）。
   > 请勿在“桌面”、“下载”或压缩包内直接运行！因为安装后该文件夹不能再被移动，否则系统将找不到执行文件。

### 2. 一键配置
1. 进入文件夹，右键或双击运行 `install.bat`。
2. 允许弹出的管理员权限请求。
3. 根据弹出的黑色窗口提示，依次输入你的【学号/工号】、【校园网密码】并选择你的【运营商】。
4. 提示“安装圆满完成”后即可关闭窗口。

*(图示：安装配置过程极简，按提示输入即可完成)*

🎉 **至此配置结束。你可以尝试断开 Wi-Fi 重新连接校园网，享受无感上网的乐趣。**

## 🗑️ 彻底卸载

如果你毕业了，或者换了电脑不再需要此功能：
1. 进入你当时存放该工具的文件夹。
2. 双击运行 `uninstall.bat`。
3. 提示卸载成功后，系统任务即被清理，此时你可以安全地将整个文件夹放进回收站。

## 👨‍💻 技术原理

本项目的核心理念是**原生与轻量**。它不依赖任何需要一直挂在后台的常驻软件（那会浪费内存），而是巧妙地利用了 Windows 系统自带的事件触发机制。

整个项目的运行分为三个明确的阶段：

### 阶段一：安装部署 (`install.bat` 做了什么？)
1. **获取权限与信息**：脚本会自动请求管理员权限，并记录你输入的账号、密码和运营商。
2. **生成发包脚本**：在当前文件夹下，它会为你单独生成一个 `login.bat` 文件，里面是一条包含了你登录信息的纯网络请求命令（curl）。
3. **注册系统规则**：这也是最核心的一步。脚本会向 Windows “任务计划程序”写入一条定制的 XML 规则，告诉系统：“以后请帮我盯着网络状态”。

### 阶段二：后台静默运行 (日常使用中发生了什么？)
为了做到真正的“无死角”登录，系统在后台布下了**三重触发网络**：
1. **网络连接触发**：当你刚插上网线或连上 Wi-Fi 的瞬间，Windows 底层会产生一个“网络已连接”的事件。规则会立刻检查连入的网络是不是 `CUMT_Stu` 或 `CUMT_Tec`，如果是则立刻激活任务。
2. **开机与唤醒保底**：如果你插着网线关机或休眠，开机时可能不会产生新的网络连接事件。为此，我们加入了**开机登录 (Logon)** 与 **睡眠解锁 (SessionUnlock)** 触发器。并且贴心地加入了 **3秒延迟**，给网卡留足分配 IP 地址的时间，完美避开了系统刚唤醒时的网络死角。
3. **系统级静默执行**：任务激活后，Windows 会动用最高级别的系统内置账户（`SYSTEM`）在底层偷偷运行 `login.bat`。由于 `SYSTEM` 账户没有可视界面，因此**全程绝对不会弹出任何黑框**。
4. **瞬间秒连**：`login.bat` 向校园网关（10.2.5.251）发送网络请求，几百毫秒内完成认证，毫不拖泥带水。

*(图示：基于 Windows 系统原生事件驱动，实现最高效的网络状态监控)*

### 阶段三：干净卸载 (`uninstall.bat` 做了什么？)
1. 从系统的“任务计划程序”中，把阶段一注册的监控规则（`CUMT_Net_AutoLogin`）彻底删除。
2. 把同目录下包含你密码的 `login.bat` 文件删除，做到真正的无痕清理。

## 💡 温馨提示（关于注销）

由于本工具会在你连接网络时自动登录，如果你遇到需要给手机或平板腾出登录名额（超出多终端限制），或是想要主动下线的情况，请在连接校园网的状态下，使用浏览器访问学校认证网关：
👉 **[http://10.2.5.251](http://10.2.5.251)**
在弹出的页面中点击“注销”按钮即可自行下线。
